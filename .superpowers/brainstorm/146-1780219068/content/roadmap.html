🏗️ Phase 1 · 基础建设（MVP 底座）

搭好骨架，让内容能存进去、能查出来

Phase 1 任务清单
1.1 项目脚手札搭建 Next.js + TypeScript + Tailwind + shadcn/ui + Supabase 初始化
1.2 数据库模型设计 User → Note → Word/Phase/Sentence → TrainingRecord → ErrorLog 表结构
1.3 用户认证 Supabase Auth 集成，邮箱/Google 登录（自用为主，简单即可）
1.4 M2 内容管理 CRUD 单词/短语/句式的手动增删改查、搜索、标签筛选、来源追溯
1.5 M1 笔记导入（四步流程） NotebookLM 连接 → 选择笔记 → Gemini 提取 → 审核确认 → 入库
1.6 基础 UI 布局 导航框架、首页 Dashboard、移动端响应式适配

🧠 Phase 2 · 核心训练（核心价值）

这就是你每天使用的核心——训练闭环

Phase 2 任务清单
2.1 M3 艾宾浩斯调度引擎 遗忘曲线算法实现（1/2/4/7/15/30天间隔），每日训练队列生成，复习优先级排序
2.2 M4 阅读训练 闪卡模式 → 翻译练习 → 完形填空 → 阅读理解，四种题型实现
2.3 M4 写作训练 中译英 + 造句 + 主题写作，Gemini API 辅助评分和纠正
2.4 M4 口语训练（文本模式） 看中文 → 说英文 → 自评打分，口语常用句式专项训练
2.5 M6 错题强化闭环 训练错题自动收录 → 短间隔重排 → 连续 3 次正确释放，错误类型归类
2.6 首次可用的端到端流程 导入 → 训练 → 复习 → 错题重做，完整闭环可用，开始产生学习价值

✨ Phase 3 · 增强体验（锦上添花）

让学习更有动力、更有反馈

Phase 3 任务清单
3.1 M5 进度统计面板 学习热力图、每日打卡、各模块掌握率、词汇量增长曲线（Chart.js）
3.2 A1 目标设定与提醒 每日学习目标设置 → 完成度追踪 → 浏览器通知推送 → 连续打卡激励
3.3 A2 Phase 1 单词发音 Web Speech API 集成，训练卡片点击发音，英美口音切换

🚀 Phase 4 · 进阶完善（长期优化）

类 App 体验 + 口语完整化

Phase 4 任务清单
4.1 A2 Phase 2 例句朗读 整句朗读、跟读文本对照、逐词高亮
4.2 A2 Phase 3 口语评测 Web Speech 录音 → Gemini AI 对比评分 → 发音纠正建议
4.3 A3 PWA 离线支持 Service Worker 缓存、离线训练、桌面安装、同步策略
4.4 最终打磨 性能优化、加载体验、错误处理完善、全流程回归测试

⚡ 关键决策点

🔀 分岔口 1：先训练还是先内容？

Phase 2 如果嫌艾宾浩斯引擎开发太重，可以先做一个简化版的固定间隔复习（每天复习前一天 + 前一周的内容），让训练闭环先跑起来，再升级为完整遗忘曲线。

简化版先跑

  • 1 天内可完成
  • 快速验证训练流程

完整引擎

  • 2-3 周打磨
  • 更科学的复习效果

🔀 分岔口 2：全部模块还是最小闭环？

Phase 2 是否同时做三种训练？还是先只做阅读（用得最多），写作和口语后续补？先做阅读可以更快出成果，但架构需预留扩展接口。
