🏗️ 英语学习网站 — 模块架构方案

推荐方案：模块化单体（Next.js 全栈），按功能边界拆分为 6 个核心模块 + 3 个辅助模块

📦 核心模块（必须实现）

📝

M1 · 笔记导入与AI提取

NotebookLM API → 内容解析 → 单词/短语/句式提取

依赖：notebooklm skill + Gemini API

M1 · 笔记导入与AI提取

对接 NotebookLM 读取笔记，AI 自动提取单词、短语、句式、语法点，按训练类型分类归档

📚

M2 · 词汇与内容管理

单词库 | 短语库 | 例句库 | 标签分类

CRUD + 搜索 + 筛选 + 批量导入

M2 · 词汇与内容管理

结构化存储所有学习内容，支持手动增删改查、按来源笔记追溯、按熟练度/标签分组

🧠

M3 · 艾宾浩斯调度引擎

遗忘曲线算法 | 复习日历 | 推送队列

核心算法模块，决定何时复习什么

M3 · 艾宾浩斯调度引擎

基于遗忘曲线计算每个学习单元的最佳复习时间点（1/2/4/7/15/30天），生成每日训练队列

🏋️

M4 · 训练执行模块

口语 | 阅读 | 写作 | 错题本

闪卡 + 翻译 + 填空 + 完形 + 写作

M4 · 训练执行模块

三种训练类型各自的多题型实现，统一的评分/纠错/错题回收机制，训练记录写入数据库

📊

M5 · 进度追踪与统计

学习日历热力图 | 掌握率 | 趋势图

Chart.js 可视化，激励持续学习

M5 · 进度追踪与统计

可视化学习数据：每日打卡热力图、各模块掌握率、复习曲线、学习时长统计、词汇量增长趋势

🔁

M6 · 错题强化闭环

错题本 | 间隔重做 | 错误类型分析

错题也按艾宾浩斯曲线重新入队

M6 · 错题强化闭环

训练中的错题自动收录，按错误类型归类，以更短间隔重新排入训练队列，直到连续 3 次正确后释放

🔧 辅助模块（建议实现）

🎯

A1 · 目标设定与提醒

每日目标 | 浏览器通知 | 连续打卡

建议补充

A1 · 目标设定与提醒

设定每日学习量（如每天 20 个单词 + 1 篇阅读），到期推送浏览器通知，连续打卡激励。低开发成本，高激励效果。

🔊

A2 · TTS语音增强

Web Speech API | 单词发音 | 例句朗读

建议补充

A2 · TTS 语音增强

利用浏览器内置 Web Speech API（零成本），为单词和例句提供发音朗读，提升听力输入。后续可升级为 AI 语音评测。

📱

A3 · PWA 离线支持

Service Worker | 离线训练 | 安装到桌面

后续增强

A3 · PWA 离线支持

将网站做成 PWA，支持离线访问已缓存内容、添加到手机桌面，实现类 App 体验。Next.js 对 PWA 有良好支持。
